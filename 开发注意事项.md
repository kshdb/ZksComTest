# 4、解决打包慢 最彻底的方法，具体操作如下：
可参考 https://blog.csdn.net/weixin_48653480/article/details/126153039
## 第一步：找到缓存目录
打包工具（如 electron-builder）默认的缓存路径通常如下：
```
Windows: C:\Users\[你的用户名]\AppData\Local\electron-builder\Cache 和 C:\Users\[你的用户名]\AppData\Local\electron\Cache

macOS: ~/Library/Caches/electron-builder 和 ~/Library/Caches/electron

Linux: ~/.cache/electron-builder 和 ~/.cache/electron
```
## 第二步：下载所需文件

你需要根据你的项目配置（Electron版本、目标平台win32/linux/darwin、架构x64/arm64）确定文件名。例如，electron-v31.0.0-win32-x64.zip。

## 从国内镜像站（如淘宝NPM镜像 https://npmmirror.com/mirrors/electron/）下载对应的文件。

## 第三步：放入缓存目录
将下载好的 .zip 文件直接放入第一步找到的对应缓存目录中。例如，electron-v31.0.0-win32-x64.zip 通常应放入 electron-builder\Cache 目录下的对应子文件夹（如 win32-x64）内。

## 第四步：下载winCodeSign
```
https://github.com/electron-userland/electron-builder-binaries/releases/download/winCodeSign-2.6.0/winCodeSign-2.6.0.7z
```

## 第五步：下载完winCodeSign 解压 复制到 C:\Users\用户\AppData\Local\electron-builder\Cache\winCodeSign
参考 https://zhuanlan.zhihu.com/p/1944764189685053383

## 第六步：下载nsis-resources https://github.com/electron-userland/electron-builder-binaries/releases/download/nsis-resources-3.4.1/nsis-resources-3.4.1.7z

## 第七步：下载完nsis 解压 复制到 C:\Users\用户\AppData\Local\electron-builder\Cache\nsis
## 第八步：下载nsis https://github.com/electron-userland/electron-builder-binaries/releases/download/nsis-3.0.4.1/nsis-3.0.4.1.7z
## 第九步：下载完nsis 解压 复制到 C:\Users\用户\AppData\Local\electron-builder\Cache\nsis


# linux 打包配置 
```
"linux": {
  "target": [
    { "target": "AppImage", "arch": ["x64", "arm64"] },
    { "target": "deb", "arch": ["x64", "arm64"] },
    { "target": "rpm", "arch": ["x64", "arm64"] }
  ]
}
```
## linux 跨架构打包 不建议 建议各自在自己的物理机上编译
